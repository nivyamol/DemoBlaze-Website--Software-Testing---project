DemoBlaze-Website--Software-Testing---project
The DemoBlaze Software Testing Project focuses on testing the functionality, usability, and performance of the e-commerce website. It includes manual and automation testing of modules like login, signup, cart, product purchase, and order placement to identify defects and ensure quality user experience using  Selenium WebDriver, Java, and TestNG 
Project Overview
This repository presents a complete software testing project executed on the Demoblaze web application. The project demonstrates both manual and automation testing practices to validate core functionalities of the system.

🔗 Application Under Test: https://www.demoblaze.com

Project Structure

Repository Contents
TEST PLAN_DEMOBLAZE (.docx)
Test planning document including scope, strategy, environment, and risks.

TESTCASES_DEMOBLAZE (.xlsx) Designed test cases covering functional, UI/UX, and non-functional scenarios.

BUG REPORT (.xlsx) Defects logged based on failed test cases with severity and priority.

UPDATED TESTCASES_REGRESSION (.xlsx) Failed test cases re-executed after fixes with updated results and status.

TEST REPORT_DEMOBLAZE (.docx) Test metrics, execution summary, defect analysis, and final closure report.
